# Spec Completeness Checklist

Mark each as `complete`, `weak`, or `missing`.

- Problem and user value are specific and measurable.
- User segments and key journeys are explicit.
- Scope boundaries prevent feature creep.
- Architecture and data contracts are concrete.
- Failure behavior is defined for degraded dependencies.
- Security and privacy constraints are testable.
- Performance targets include measurable thresholds.
- Rollout/rollback strategy is executable.
- Acceptance criteria are objective and verifiable.
- Major tradeoffs include rationale and rejected options.
